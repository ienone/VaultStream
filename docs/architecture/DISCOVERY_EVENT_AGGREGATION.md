# 📂 Discovery 事件级聚合与动态分组设计方案

## 1. 概述
本方案旨在解决发现流中的内容重合问题，通过 AI 聚类与大模型综述能力，将指向同一事件的不同来源（URL）动态聚合。提供类似“浏览器标签页分组”的交互体验，支持手动拖拽管理，并针对 Token 成本与响应速度进行了深度优化。

---

## 2. 核心交互设计 (UI/UX)

### 2.1 双态呈现逻辑
系统默认保持轻量，仅在检测到重合或用户操作时转化形态：
- **独立模式 (Independent)**：普通资讯显示为单张简洁卡片。
- **聚合组模式 (Aggregated)**：当内容重合时，卡片演变为“堆叠分组”样式。

### 2.2 交互规范 (Browser-Tab Style)
- **列表态**：
    - **父卡片**：显示由 LLM 生成的【事件综述标题】，左下角标注来源数量。
    - **展开交互**：点击展开按钮，主卡片下方滑出各来源的子卡片（真实标题与互动数据）。
- **详情联动**：
    - 点击父卡片：右侧展示 LLM 综述的事件内容、包含的图片等。
    - 点击子卡片：右侧展示该来源的原文、附件。
- **管理交互 (Drag & Drop)**：
    - **合并 (Merge)**：长按卡片 A 拖拽至卡片 B，触发合并并异步生成综述。
    - **拆分 (Split)**：在展开的组内，长按子卡片拖出组外，该项恢复为独立卡片。

---

## 3. 后端存储与数据模型

### 3.1 树状结构实现
在 `contents` 表中通过递归外键支持父子关系：
- 父节点的原文链接存储各来源URL,而非单一url。
- `parent_id` (Integer): 指向父级 `Content.id`。独立项为空。
- `is_synthesis` (Boolean): 标记是否为 AI 生成的虚拟综述节点。
- **父节点特征**：`platform="universal"`, `discovery_state="merged"`, `is_synthesis=true`。其标题与摘要由 LLM 综合生成。

### 3.2 成员清单 (Manifest)
父节点的 `context_data` 存储 `synthesis_memo`，记录：
- `member_ids`: 组内成员列表及加入顺序。
- `version`: 综述更新版本号。
- `last_synthesis_at`: 上次生成时间。

---

## 4. 性能与 Token 优化策略

### 4.1 增量综述算法 (Incremental Synthesis)
为了节约 Token 并提高合并速度，系统不执行“全量重算”：
- **追加聚合**：当新项 C 并入 [A, B] 时，LLM 输入 = `(现有综述 V1 + 项 C 摘要)`。
- **Prompt 示例**：基于现有综述更新信息，而非重新阅读所有原文。

### 4.2 摘要缓存 (Summary Caching)
- 所有内容入库时强制生成并持久化 `summary`。
- 聚合、重组、拆分操作均仅读取 `summary` 字段，避免读取 `body` 长文本，Token 消耗降低 >80%。

### 4.3 异步响应机制
- **流程**：`API (202 Accepted) -> 消息队列 -> LLM 综述任务 -> 数据库更新 -> SSE 推送`。
- **前端表现**：拖拽完成后卡片立即进入“AI 处理中”占位态，综述生成后标题自动淡入更新。

---

## 5. 特殊场景处理 (Edge Cases)
- **溶解 (Dissolve)**：当拆分操作导致聚合组内仅剩 1 个子项时，系统自动删除父节点，将子节点恢复为独立 `visible` 状态。
- **多轮合并**：支持将一个聚合组拖入另一个聚合组，后端执行平铺展开后重组。
